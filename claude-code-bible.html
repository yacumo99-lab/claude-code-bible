사전준비Node.js 설치 확인 (필수!)
▼
왜 Node.js가 필요한가?
이유클로드 코드 자체가 Node.js 기반으로 만들어진 프로그램이에요. Node.js가 없으면 설치 자체가 안 돼요.
버전Node.js 18 이상이 필요해요. 낮은 버전이면 오류가 나요.
현재 버전 확인
$node --version
v20.0.0 처럼 나오면 정상. v18 미만이면 nodejs.org에서 새 버전 설치
$npm --version
Node.js 설치하면 npm은 자동으로 같이 설치돼요!
설치클로드 코드 설치 · 업데이트 · 제거
▼
npm install -g 란?
-gglobal(전역)의 줄임말. 내 컴퓨터 어디서나 claude 명령어를 쓸 수 있게 설치해줘요.
@@anthropic-ai/claude-code 는 패키지 이름이에요. npm 저장소에서 자동으로 내려받아요.
처음 설치 (최초 1회)
$npm install -g @anthropic-ai/claude-code
최신 버전으로 업데이트
$npm update -g @anthropic-ai/claude-code
새 기능이 추가되거나 버그가 고쳐지면 업데이트하면 돼요.
설치된 버전 확인
$claude --version
완전 제거 (삭제)
$npm uninstall -g @anthropic-ai/claude-code
실행클로드 코드 실행 방법 전체
▼
기본 실행 — 대화형 모드
$claude
대화형 모드란?
의미클로드가 계속 켜져 있고, 내가 질문하면 바로 답해주는 채팅 방식이에요. 일반적으로 가장 많이 쓰는 방식.
특정 파일을 열면서 시작
$claude 파일명.py
클로드가 시작하면서 그 파일을 자동으로 읽어둬요. "이 파일 어때?" 바로 물어볼 수 있어요.
질문 하나만 하고 바로 종료 (비대화형)
$claude -p "이 코드 분석해줘"
-p 옵션이란?
의미--print 의 줄임. 대화 창 없이 질문 하나 던지고 답 받고 바로 종료해요. 자동화 스크립트에서 많이 써요.
파일 내용을 입력으로 넘기기
$cat 파일명.py | claude -p "설명해줘"
cat 으로 파일을 읽어서 | (파이프)로 클로드에 전달해요. 파일을 직접 첨부하지 않아도 돼요.
작업 폴더 지정하면서 시작
$claude --cwd /경로/폴더명
터미널터미널 기본 명령어 완전 정리
▼
폴더 이동 (cd = Change Directory)
$cd 폴더이름
$cd .. ← 상위 폴더(부모)로
$cd ~ ← 홈 폴더 (내 사용자 폴더)
$cd / ← 루트(최상위) 폴더
현재 위치 확인 (pwd = Print Working Directory)
$pwd ← 맥/리눅스
$cd ← 윈도우 (파라미터 없이)
파일 목록 보기 (ls / dir)
$ls ← 맥/리눅스 기본
$ls -la ← 숨김파일 포함 + 자세한 정보
$dir ← 윈도우
폴더 만들기 (mkdir = Make Directory)
$mkdir 폴더이름
$mkdir -p 상위/중간/하위 ← 중간 폴더도 자동 생성
파일 내용 출력 (cat)
$cat 파일명.txt
파일 복사 / 이동 / 삭제
$cp 원본.py 복사본.py
$mv 파일.py 새이름.py ← 이름 변경도 가능
$rm 파일.py ← 파일 삭제
rm 은 복구 불가! 삭제 전 확인 필수!
파이썬 실행
$python 파일명.py ← 윈도우
$python3 파일명.py ← 맥/리눅스
pipPython 패키지 관리 (pip) 완전 정리
▼
pip 이란?
정의Python Package Installer. 파이썬에서 쓸 수 있는 외부 라이브러리(패키지)를 인터넷에서 자동으로 내려받아 설치해줘요.
예시pandas (데이터 분석), requests (웹 요청), openpyxl (엑셀) 등을 pip으로 설치해요.
$pip install 패키지이름
$pip install pandas requests openpyxl ← 여러 개
$pip install -r requirements.txt ← 목록 파일로 한번에
$pip install --upgrade 패키지이름 ← 업데이트
$pip uninstall 패키지이름
$pip list ← 설치된 패키지 전체
$pip show 패키지이름 ← 특정 패키지 상세 정보
$pip freeze > requirements.txt ← 현재 패키지 목록 저장
pip freeze > requirements.txt 하면 나중에 pip install -r requirements.txt 로 한방에 재설치 가능!
venv가상환경 (venv) — 왜 써야 하나?
▼
가상환경이 왜 필요한가?
문제프로젝트 A는 pandas 1.0, 프로젝트 B는 pandas 2.0이 필요할 때 — 둘을 같이 쓰면 충돌나요.
해결프로젝트마다 독립된 파이썬 환경(가상환경)을 만들면 각각 다른 버전을 쓸 수 있어요.
비유가상환경 = 각 프로젝트만을 위한 전용 방. 다른 방 물건이 섞이지 않아요.
① 가상환경 만들기 (최초 1회)
$python -m venv venv
프로젝트 폴더 안에 'venv'라는 폴더가 생겨요. 이 안에 독립된 파이썬이 들어있어요.
② 가상환경 켜기 (매번 작업 시작할 때)
$venv\Scripts\activate ← 윈도우
$source venv/bin/activate ← 맥/리눅스
켜지면 터미널 앞에 (venv) 표시! 이 상태에서 pip install 해야 가상환경에 설치돼요.
③ 가상환경 끄기
$deactivate
venv/ 폴더는 .gitignore에 추가해서 깃허브에 올리지 마세요! 용량이 엄청 커요.

핵심개념CLAUDE.md 란 무엇이고 왜 써야 하나?
▼
CLAUDE.md 의 역할
정의클로드 코드가 시작할 때 자동으로 읽는 "프로젝트 설명서" 파일이에요. 마치 새 직원에게 주는 업무 매뉴얼 같은 거예요.
효과매번 "이 프로젝트는 파이썬이고, 주석은 한국어로 써줘..." 라고 설명하지 않아도 돼요. 한번 적어두면 자동으로 기억해요.
팀작업팀원들이 모두 같은 CLAUDE.md를 쓰면 클로드가 일관된 방식으로 작업해줘요.
자동 생성 (클로드가 분석해서 초안 작성)
/init
새 프로젝트 만들면 /init 부터! 클로드가 프로젝트를 분석해서 CLAUDE.md 초안을 자동으로 써줘요.
위치CLAUDE.md 파일 위치 — 어디에 놓느냐에 따라 범위가 달라요
▼
위치적용 범위 & 설명
~/.claude/CLAUDE.md 🌍 모든 프로젝트에 공통 적용
이름, 선호 언어, 항상 적용할 규칙 등 공통 설정을 여기에! 예: "항상 한국어 주석"
프로젝트폴더/CLAUDE.md 📁 해당 프로젝트에만 적용
프로젝트별 특수 규칙, 기술 스택, API 정보 등. 가장 많이 쓰는 방식.
src/CLAUDE.md 📂 해당 하위 폴더 작업 시 추가 적용
특정 모듈만의 규칙이 있을 때.
홈 폴더 CLAUDE.md + 프로젝트 CLAUDE.md 를 함께 쓰면 최강 조합!
작성예시CLAUDE.md 실전 작성 예시 (바이블코딩 특화)
▼

개념MCP 서버란 무엇인가?
▼
MCP (Model Context Protocol) 설명
쉬운설명클로드에게 "손과 발"을 달아주는 것이에요. 기본 클로드는 대화만 할 수 있지만, MCP를 연결하면 실제 외부 시스템에 접근할 수 있어요.
예시파일시스템 MCP → 내 PC 파일 읽기/쓰기, GitHub MCP → 코드 저장소 관리, DB MCP → 데이터베이스 쿼리 실행
  • 📁
    파일시스템 — 지정된 폴더 파일 접근 권한 확장
  • 🐙
    GitHub — PR 생성, 코드 리뷰, 이슈 관리
  • 🗄️
    SQLite/DB — 데이터베이스에 직접 쿼리
  • 🔍
    Brave Search — 인터넷 검색 기능 추가
  • 📝
    Notion/Slack — 협업 도구 연동

개념에이전트 모드란?
▼
에이전트 vs 일반 대화의 차이
일반대화내가 묻고 → 클로드가 답하고 → 내가 다음 질문. 한 번에 하나씩.
에이전트내가 목표를 주면 → 클로드가 스스로 계획 → 여러 단계를 자동으로 순서대로 실행. 나는 결과만 확인!
비유일반: 직접 요리 → 에이전트: 요리사에게 "오늘 저녁 준비해줘" 하고 맡기는 것
  • 🧠
    오케스트레이터 — 전체 작업을 계획하고 서브에이전트들에게 지시
  • ⚙️
    서브에이전트 — 실제로 파일 수정, 명령 실행, 테스트 등을 담당
  • 🔄
    복잡한 작업을 병렬로 처리해서 빠르게 완료 가능
실행에이전트 모드 실행 방법

commit바구니 안의 파일을 실제로 저장. "세이브 포인트" 만들기.

브랜치브랜치 관리 (안전한 작업 공간)
▼
브랜치란?
비유원본 코드를 복사해서 실험하는 별도 공간. 실험이 성공하면 원본에 합치고, 실패하면 그냥 버려요.

$git branch ← 브랜치 목록
$git checkout -b 새브랜치이름 ← 생성 + 이동
$git checkout 브랜치이름 ← 브랜치 이동
$git merge 합칠브랜치 ← 브랜치 합치기
$git branch -d 브랜치이름 ← 브랜치 삭제

AttributeError: 'NoneType' has no attribute
None(빈 값)인 변수에서 메서드를 호출한 것. 데이터가 없을 때 자주 발생.

TypeError: unsupported operand type
타입이 맞지 않는 연산. 예: 문자열 + 숫자, int와 str 비교.

FileNotFoundError: No such file or directory
파일 경로가 잘못됐거나 파일이 없는 것.
해결: os.path.exists(경로) 로 실제 존재 확인 후 경로 수정
ConnectionError / TimeoutError / requests.exceptions
인터넷 연결 문제 또는 API 서버 응답 없음.
해결: 인터넷 확인, API 키 확인, 잠시 후 재시도
