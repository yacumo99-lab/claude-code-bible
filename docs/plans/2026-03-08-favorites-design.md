# 즐겨찾기 기능 디자인

**날짜:** 2026-03-08
**상태:** 승인됨

## 목표

자주 쓰는 명령어를 개별 저장하고, 상단 고정 패널에서 빠르게 접근 및 복사할 수 있게 한다.

## UI 구조

```
[탭바]
[⭐ 즐겨찾기 ▼]  ← 접기/펼치기 헤더 (비어있으면 숨김)
┌─────────────────────────────┐
│ claude --model opus  [복사] [✕] │
│ /clear               [복사] [✕] │
└─────────────────────────────┘
[섹션 콘텐츠]
```

## 동작 방식

### 명령어 줄 변경
- 모든 `.cmd` 줄에 `⭐` 버튼 추가 (복사 버튼 오른쪽)
- ⭐ 클릭 → 즐겨찾기 추가, 버튼이 `★` (노란색)으로 변함
- ★ 클릭 → 즐겨찾기 제거, 버튼이 `⭐`로 돌아감

### 즐겨찾기 패널
- 위치: 탭바 바로 아래, 섹션 콘텐츠 위
- 기본 상태: 펼쳐짐 (즐겨찾기가 1개 이상일 때만 표시)
- 헤더 클릭으로 접기/펼치기
- 각 항목: 명령어 텍스트 + [복사] + [✕ 제거]
- ✕ 클릭 시 패널에서 제거 + 원본 카드의 ★도 ⭐로 해제

### 데이터 저장
- `localStorage["bible-favs"]` 에 명령어 문자열 배열 저장
- 예: `["claude --model opus", "/clear"]`
- 앱 재시작 후에도 유지

## 신규 JS 함수

| 함수 | 역할 |
|------|------|
| `loadFavs()` | 앱 시작 시 localStorage에서 불러와 패널 렌더 + ★ 버튼 동기화 |
| `toggleFav(btn)` | ⭐/★ 버튼 클릭 시 추가/제거 토글, 저장, 패널 갱신 |
| `removeFav(cmd)` | 패널 내 ✕ 클릭 시 제거, 원본 버튼 상태 동기화 |
| `renderFavPanel()` | 패널 HTML 전체 다시 그리기 |

## 영향 범위
